## Overview

ยกระดับ **GKS access architecture** จาก RAG-style → 4-layer system (truth + providers + protocol + agents) โดย:
1. ขุด abstraction layer (`RetrievalProvider`) ระหว่าง MSP กับ storage
2. ทำ 4 provider: Atomic / FTS / Vector / Graph
3. สร้าง indexer pipeline ที่ rebuild ทุกอย่างจาก truth (markdown + jsonl)
4. เปิด `context.resolve()` API แทน `retrieve()` เดิม
5. Formalize knowledge gate (inbound → promote → gks/)

**ไม่ใช่ rewrite** — ต่อยอด MVP ที่ merge ไปแล้ว (PR #2) โดย keep backward compat 1 release

## Phased Waves

### Wave 0 — Foundation (abstraction, no behavior change)
**Goal**: interface มาก่อน implementation
- T1: Define `RetrievalProvider` interface + Query/Hit types
- T2: Refactor `MemoryStore` to delegate to providers (keep old API as facade)

### Wave 1 — Providers (each independently testable)
**Goal**: ทุก provider ใช้งานเดี่ยวได้
- T3: `AtomicIndexProvider` (O(1) exact lookup)
- T4: `RipgrepFtsProvider` (spawn rg + parse)
- T5: `FileVectorProvider` refactor (existing VectorStore → provider shape)
- T6: `BacklinkGraphProvider` (load `.eva/vector/backlinks.jsonl`)

### Wave 2 — Orchestrator (hybrid logic)
**Goal**: รวม providers ผ่าน HybridRetriever
- T7: `HybridRetriever` cascade + budget
- T8: RRF reranker + boost logic
- T9: `context.resolve()` API on MSP facade

### Wave 3 — Indexer (truth → indexes)
**Goal**: rebuild ทุก index จาก markdown truth
- T10: `scripts/msp/index.mjs` — parse + chunk + extract + embed + push
- T11: Frontmatter crosslink parser → backlinks/backref jsonl
- T12: Add `package.json` script `"msp:index"`

### Wave 4 — Governance (knowledge gate)
**Goal**: formal propose → review → commit flow
- T13: `scripts/msp/propose.mjs` (agent CLI to drop into inbound)
- T14: `scripts/msp/promote.mjs` (human CLI: inbound → gks/)
- T15: Review state in inbound frontmatter + tracker

### Wave 5 — Integration + migration
**Goal**: wire into existing orchestrator + deprecate old paths
- T16: AgentLoop: migrate from `retrieve()` to `resolveContext()`
- T17: Deprecate `MemoryStore.retrieve()` (JSDoc + console.warn once)
- T18: Smoke test — run agent → real query → verify hits
- T19: Docs update (EVA.md, README)

## Risks & Mitigation

| Risk | Severity | Mitigation |
|---|---|---|
| Refactor breaks existing agent | HIGH | Keep `retrieve()` as facade calling new API; run full test suite after each wave |
| Indexer pipeline เช้าช้า (> 2 min) | MED | Batch embed, parallelism, incremental mode (only re-embed changed files via hash) |
| ripgrep not available in prod | LOW | Fallback SQLite FTS5 in T4 |
| Graph cycles crash traversal | LOW | Depth limit + visited set in BacklinkGraphProvider |
| Backlinks file corrupt | LOW | Validate on load; rebuild from source on failure |
| Vector search precision ต่ำหลัง rerank | MED | A/B test: old `retrieve()` vs new `resolveContext()` on 20 Thai queries, human eval |
